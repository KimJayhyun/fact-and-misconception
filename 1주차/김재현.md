## 서문

객체지향의 특징

- 추상화, 캡슐화, 상속, 다형성: 다른 패러다임과 구분하는 차이점
- SOLID & 디자인 패턴

객체 지향을 이해하기 위한 방향

1. 클래스가 아닌 객체를 보라
2. 객체를 독립적인 존재가 아닌 기능을 구현하기 위해 협력하는 공동체의 존재로 보라
3. 협력에 참여하는 객체에 적절한 역할과 책임을 부여할 수 있는가
4. 이해한 개념들을 프로그래밍 언어에 담아낼 수 있는 기술 익히기

## 책의 구성

### 1. 협력하는 공동체

객체지향의 핵심은 자율적인 객체들의 협력

- 객체지향의 중심은 클래스가 아닌 객체

### 2. 이상한 나라의 객체

객체란 무엇인가

- 상태와 행동, 식별자를 가진 존재

### 3. 타입과 추상화

추상화 = 단순화

- 동적인 객체를 단순화해 정적인 타입으로 단순화

타입에 대한 설명과 객체를 클래스로 구현하는 이유

### 4. 역할, 책임, 협력

객체지향의 핵심인 역할, 책임, 협력에 대한 설명

### 5. 책임과 메세지 - 핵심

훌륭한 메세지 = 훌륭한 객체지향 설계의 기반

객체의 자율성과 설계의 유연성은 메세지에 달려있음

### 6. 객체 지도

객체지향이 구조와 기능을 조화시키는 방법

- 구조는 기능에 비해 변화에 안정적
- 구조 안에 기능을 녹임으로써 변화에 안정적인 소프트웨어 개발

## Chapter 2. 이상한 나라의 객체

아기가 인지하는 사물의 범위

### 객체지향과 인지능력

실세게는 물질적인 사물뿐만 아니라 추상적인 사물로 이루어져있고,

사람은 두 가지 모두 객체로 인지한다.

더 작은 객체로 나누는 것은 좀 더 단순한 객체로 구성하여 복잡성을 극복하기 위함

즉, 객체란 인간이 분명히 인지하고 구별할 수 있는 물질적 혹은 개념적인 경계

객체지향 프로그래밍 또한 소프트웨어 세계엔 인간이 인지할 수 있는 소프트웨어 객체로 이루워져있다고 맏음

- 실세계와의 유사성은 여기까지
- 현실의 전등은 불을 스스로 끄고 킬 수 없지만, 주문 객체는 스스로 계산을 한다.

### 객체 그리고 이상한 나라

#### 엘리스 객체

엘리스의 상태를 결정하는 것은 행동이지만, 행동의 결과를 결정하는 것은 상태이다.

행동의 성공 여부는 이전의 행동들이 발생했는지도 영향을 받음

- 행동의 순서도 중요하다.

엘리스의 특성

1. 상태를 가지며 상태는 변경 가능하다.
2. 상태를 변경시키는 것은 행동이다.
   - 행동의 결과는 상태에 의존적이며 상태를 이용해 서술할 수 있다.
   - 행동의 순서가 결과에 영향을 미친다.
3. 어떤 상태에 있더라도 유일하게 식별 가능하다.

### 객체, 그리고 소프트웨어 나라

객체는 상태, 행동, 식별자를 지닌 실체로 보는 것이 설명하기 가장 효과적이다

객체의 정의

    식별가능한 개체 또는 사물
    구체적이거나 추상적일 수 있음
    식별자, 행동, 상태를 가짐
    소프트웨어 안에서는 저장된 상태와 실행 가능한 코드로 구현됨

#### 상태

> 왜 상태가 필요한가.

객체가 주변 환경과 상호작용하기 위해서는 객체에게 어떤 일이 발생했는가에 좌우된다.

객체에게 일어난 모든 행동을 통해 다음 행동의 결과를 판단하기에는 복잡하고 번거롭고 이해하기 힘들다.

- 과거에 의존하면 어렵다.

행동의 결과를 상태로 나타내면 이해하기 쉽다.

- 현재를 기반으로 행동 방식 등을 이해할 수 있다.

> 상태와 프로퍼티

객체의 상태는 단순한 값과 객체의 조합으로 표현할 수 있다.

객체의 상태를 구성하는 모든 특징을 객체의 프로퍼티(Property)라고 한다.

property는 변하지 않으므로 정적이지만, property value는 동적이다.

객체와 객체사이의 의미있는 연결을 Link라고 한다.

링크를 통해서만 메세지를 주고 받는다.

객체를 구성하는 단순한 값은 속성(attribute)라고 한다.

객체의 프로퍼티는 링크와 속성으로 이루어져있다.

상태의 정의

    특정 시점에 객체가 가지고 있는 정보의 집합
    객체의 상태는 객체에 존재하는 프로퍼티와 프로퍼티 값으로 구성된다.
    객체의 프로퍼티는 단순한 값과 다른 객체를 참조하는 링크로 구분된다.

객체는 자율적인 존재이므로 다른 객체의 상태에 직접적으로 접근, 변경할 수 없다.

간접적으로 접근, 변경할 수 있는 방법이 필요하다. => 행동
